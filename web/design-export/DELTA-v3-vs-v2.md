## C. 弹窗(README 只提了 C-3,其余都没写)

### C-1 🆕 全屏订阅页 `subOverlay()` / `subCard()`（304–385）— 完全新增
- 顶部促销条「Seedance 2.0 低至 0.38 元/秒」+ 标题「选择适合你的订阅方案」
- 切换:**个人版 / 企业版(New)** × **按月 / 按年(省 40%+)**
- 「我的积分 8,842 + 购买积分」入口 + 「企业定制方案 →」
- **8 个套餐**(个人 4 + 企业 4),数据在 `SUB_PLANS`(305–317):
  - 个人:BASE/STAR(最受欢迎)/PRO/APEX,¥83~999/月起,1000~18000 Credits/月
  - 企业:TEAM LITE/PLUS/PRO/MAX,4000~46000 Credits/月,含团队席位/共享等专属权益
- 每卡展示:折扣% / 原价划线 / 年付折算 / Credits/月 + 约 N 张图·N 秒视频 / 模型权益(Seedance 2.0、HappyHorse、N 路并发)/ 无水印·1080p 等
- ⚠️ 这是最大的一块新增,涉及完整定价体系,建议产品先敲定套餐数据

### C-2 ✏️ 账户/余额弹窗 `acctModal()`（126–219）
- v2.0 是独立页面 `/account`(AccountPage)。设计稿改成**弹窗**,左侧 tab:`账户与计费` / `兑换邀请码`
- 余额明细卡:通用积分/专属积分(可用+冻结)、订阅/充值/奖励 三类拆分、`当前为 FREE + 升级获取积分`
- 下方 `积分明细·所有充值` / `订阅与账单` 两个表 tab + 下载发票/订单反馈
- 🆕 `兑换邀请码` tab:输入框 + 确认,文案"绑定邀请码获得积分"
- ⚠️ 要决定:是把 `/account` 页改成弹窗,还是两者并存

### C-3 ✏️ 积分充值弹窗 `rechargeModal()`（221–271）— README 主讲
- v2.0 已有 `RechargeDialog.tsx`,但这次**触发点、布局、套餐都改了**:
  - 触发:用户菜单「购买积分」`window.zyOpenRecharge()`(原来点头像余额)
  - 用户卡(头像 + FREE + 升级)+ 右侧积分余额
  - **7 个套餐**:490/1400/2100/3500/7000/14000(¥49~1400)+ 1 个宽卡 `70,000→84,000`(会员额外赠送 20%,¥7000)
  - 选中态:accent 边框 + soft 底 + `0 0 0 3px oklch(76% .13 70/.26)` 光晕,默认选第一个
  - 提示文案(会员权益/积分有效期 1 年·赠送 31 天)+ 底部「购买积分」→ 开支付弹窗
  - 响应式 ≤600px 变 2 列

### C-4 ✏️ 支付方式弹窗 `payModal()`（273–302）

## D. 🆕 新页面 `admin-usage.html` 「所有消耗」（你标红的②的落地页,245 行）— 完全新增

平台管理员专用的**全平台积分消耗审计页**,v2.0 的 4 个 admin 页都没有这个。
- 标题「所有消耗」+ 副标题"平台下所有账户的积分消耗记录,可定位每条消耗的主账号/子账号"
- 3 统计卡:**今日消耗 / 本月消耗 / 活跃子账号**
- 筛选区 `.filter-grid`:时间区间、主账号、子账号、任务 ID、状态(成功/失败)、清晰度、生成类型(常规/首尾帧/智能多帧)、消耗区间;底部 共 N 条 / 重置 / 导出 CSV / 官方单价 / 总成本公式
- 大表 14 列:时间·主账号·子账号·类型·清晰度·任务ID·状态·**消耗 token(千)**·**官方单价(元/千token)**·总成本·**消耗 Credit**·消耗金额·**消耗快币**·详情

## 落地建议(优先级)

1. **先定数据契约**:订阅套餐表(C-1)、积分套餐表(C-3)、快币/Credit 换算、每日奖励规则、消耗记录字段(D)——这些都要后端接口,建议先和后端对齐 `/api/v1` 契约。
2. **纯前端可先做的**:顶栏 credit-pill(A)、用户菜单头部积分区(B)、积分充值弹窗(C-3,已有 RechargeDialog 可改造)、所有消耗页(D,纯展示+筛选)。
3. **样式**:设计稿所有颜色都用了你现有的 token(accent 琥珀金、dark/light),`app.css` 里新增了 `.um-* .rc-* .sub-* .credit-pill` 等 class,落地时映射到 React 组件即可,不要硬编码颜色。
